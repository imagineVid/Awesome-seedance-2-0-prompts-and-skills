# Contributing to the Seedance 2.0 collection

Submit a case only when the canonical post explicitly identifies Seedance 2.0, contains a reusable prompt, attributes the creator, and exposes inspectable video result media.

## Review gates

1. Provenance: canonical post and creator are preserved.
2. Prompt completeness: enough text is available to reproduce the production logic.
3. Model evidence: the post names Seedance 2.0; inference is not accepted.
4. Media evidence: the result can be inspected from the repository.
5. Originality: duplicate sources, prompts, and media are rejected.
6. Editorial value: the case teaches a transferable workflow rather than merely promoting a tool.
